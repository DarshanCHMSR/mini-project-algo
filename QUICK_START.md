# SE-GradCAM TCN Quick Start Guide

## Installation & Setup

```bash
# Install dependencies
python run_training.py --install-missing

# Or manually
pip install numpy pandas pyarrow scikit-learn torch matplotlib

# Verify dataset is in place
ls dataset_V2.parquet
```

## Training the Model

```bash
# Standard training with explainability analysis
python run_training.py

# Or directly (after dependency checks)
python train.py
```

**Output:**
- Terminal output showing 5-fold CV metrics and test evaluation
- `best_model.pt`: Best TCN model weights
- `norm_stats.pkl`: Normalization statistics
- `raw_ts_cache.pkl`: Raw time-series cache
- `saliency_cycle_*.png`: Saliency plots for all defective test cycles

**Runtime:** ~10-30 minutes depending on GPU availability

## Explaining a Single Cycle

```bash
python explain_single_cycle.py "CYCLE_12345"
```

Replace `"CYCLE_12345"` with actual MET_MachineCycleID from your dataset.

**Requirements:**
- `norm_stats.pkl` (generated by train.py)
- `best_model.pt` (generated by train.py)
- `raw_ts_cache.pkl` (generated by train.py)

**Output:**
- Saliency plot saved as `saliency_cycle_CYCLE_12345.png`
- Console summary with prediction and true label

---

## Understanding the Output

### Saliency Plot (3 Panels)

**Panel 1: Pressure Signal**
- Raw DXP_Inj1PrsAct (bar) over time
- Phase backgrounds:
  - **Blue**: Injection phase (material filling mold)
  - **Orange**: Holding phase (pressure maintenance)
  - **Green**: Cooling phase (solidification)

**Panel 2: GradCAM Saliency**
- Red shaded area shows where the model attended
- Higher values = model paid more attention
- Same phase backgrounds for reference
- Normalized to [0, 1] for cross-cycle comparison

**Panel 3: SE Channel Weights (if available)**
- Grouped horizontal bars, one set per TCN block
- Shows which sensors each block weighted most
- 8 channels: Pressure, Position, Temp (2x), DOS Rate, 3x Triggers
- Colors: Different color per block for easy comparison

---

## Interpreting Results

### What Does High Saliency Mean?
- Model detected abnormality at that time point
- For defective parts: timestamp where defect likely originated
- For OK parts: normal variation the model learned to distinguish

### Phase Analysis
- "Model primarily attended to Holding phase" → defects develop during pressure maintenance
- "Injection phase" → problems during mold filling
- "Cooling phase" → issues during solidification

### Channel Weights
- High weight = sensor important for this decision path
- Pressure high in all blocks = pressure is universal defect indicator
- Temperature spike → temperature-related defect mechanism

## Modifying Hyperparameters

All training configs in `train.py` near line 45:

```python
# Model architecture
IN_CHANNELS = 8
CHANNEL_LIST = [64, 64, 128]  # Feature channels per TCN block
KERNEL_SIZE = 3
N_SCALAR_FEATS = 5
DROPOUT = 0.1

# Training
LEARNING_RATE = 1e-3
WEIGHT_DECAY = 1e-4
BATCH_SIZE = 32
MAX_EPOCHS = 100
EARLY_STOP_PATIENCE = 15

# Loss (SE-GradCAM uses AsymmetricFocalLoss now)
# gamma_pos=4.0 forces focus on hard-to-detect defects
# gamma_neg=2.0 standard focal for OK parts
```

SE attention reduction factor in `tcn_model.py`:
```python
self.se = SqueezeExcitation(out_channels, reduction=4)  # bottleneck = channels/4
```

## Troubleshooting

### Error: "norm_stats.pkl not found"
→ Run `train.py` first to generate statistics

### Error: "best_model.pt not found"
→ Run `train.py` first to train and save model

### Error: "MET_MachineCycleID column not found"
→ Dataset column name mismatch; check your parquet structure

### Matplotlib "No display" error
→ Add before running: `import matplotlib; matplotlib.use('Agg')`

### CUDA out of memory
→ Reduce BATCH_SIZE in train.py (line ~52)
→ Or use `DEVICE = torch.device('cpu')` to force CPU

## Architecture Summary

**SE-GradCAM TCN** = SE (channel attention) + GradCAM (temporal saliency)

**SE Block** (per TCN layer):
- Learns which sensors matter most
- Output: per-channel weights [0, 1]

**GradCAM** (final TCN block):
- Computes gradient × feature map
- Shows timestep importance
- Output: per-timestep saliency [0, 1]

**Asymmetric Focal Loss**:
- gamma_pos=4: Penalize missed defects heavily
- gamma_neg=2: Standard weighting for OK parts
- Result: Model focuses on hard-to-detect defects
